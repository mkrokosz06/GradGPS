Scrape bulletins.psu.edu · parse requirement structure · output clean dataset

  • ✓
    Requirement group parser — walks each program page HTML, classifies headings into requirement groups, detects group type from heading text
  • ✓
    Four group type detection — required, choose_one, choose_credits, choose_courses classified per heading text pattern
  • ✓

  • ✓

  • ✓

required
Student must complete every course listed. Audit: each course must appear on transcript.
choose_one
Courses share a pair_group_id. Audit: at least one course per pair_group_id must be on transcript.
choose_credits
Take any courses from the list until group_threshold credits are reached.
choose_courses
Take exactly group_threshold courses from the list (any combination).
Output: scrape_psu.py → PSU_Major_Requirements.xlsx (31,734 rows · 551 programs) Columns: program_name, college, degree, campus, requirement_group, group_type, group_threshold, course_code, course_title, credits, min_grade, pair_group_id, url

2B — Transcript Parser
  • →
    pdfplumber extraction — open unofficial PSU transcript PDF, extract text line-by-line, parse course codes + grades + terms with regex
  • →
    In-progress detection — rows with attempted > 0 and earned = 0 and no grade = currently enrolled, flag as in_progress
  • →
    Transfer credit handling — rows marked "TR" grade get status transfer, counted as complete for audit purposes
  • →
    Store to transcript_courses table — upsert on (user_id, course_code, term) so re-uploads cleanly overwrite
def parse_transcript(pdf_bytes: bytes) -> list[dict]: with pdfplumber.open(io.BytesIO(pdf_bytes)) as pdf: text = "\n".join(p.extract_text() for p in pdf.pages) # Match: DEPT 123 Description 3.000 3.000 A 12.000 pattern = r"([A-Z]{2,6})\s+(\d{3}[A-Z]?)\s+.+?\s+(\d+\.\d+)\s+(\d+\.\d+)\s+([A-Z][+-]?|TR)\s" courses = [] for m in re.finditer(pattern, text): earned = float(m.group(4)) grade = m.group(5) status = "transfer" if grade == "TR" else "done" if earned > 0 else "in_progress" courses.append({"course_code": f"{m.group(1)} {m.group(2)}", "grade": grade, "credits_earned": earned, "status": status}) return courses
2C — Audit Logic
  • →
    required group — for each course in group: check if course_code is in student's done/in_progress list → done / in_progress / missing
  • →
    choose_one group (pair_group_id) — group requirement rows by pair_group_id; for each pair: if any course in pair is done → satisfied; if any is in_progress → in_progress; else → missing
  • →
    choose_credits group — sum credits earned from courses in the pool that appear on transcript; if sum >= group_threshold → satisfied; else show remaining credits needed
  • →
    choose_courses group — count distinct courses from pool on transcript; if count >= group_threshold → satisfied
  • →
    Min grade check — if min_grade is set, a course only counts as "done" if the student's grade meets or exceeds the requirement (C or better, etc.)
